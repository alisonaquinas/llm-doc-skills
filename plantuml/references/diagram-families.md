# PlantUML Diagram Families

PlantUML covers a wide set of diagram families.
Select the one that matches the system question instead of building an all-purpose diagram.

## Common families

- class diagrams for structural type relationships
- sequence diagrams for time-ordered interaction
- activity diagrams for flow and control paths
- component and deployment diagrams for runtime structure
- state diagrams for lifecycle modeling
- use case, object, timing, and other specialized views when they fit

## Working guidance

Keep one primary narrative per diagram.
If structural and behavioral views are both important, split them into separate diagrams.
Use aliases and consistent naming so long labels do not dominate the layout.

## Review checklist

- confirm the chosen family matches the intended reasoning task
- confirm labels and notes do not overwhelm the diagram body
- confirm relationships are directional and readable without surrounding explanation
- confirm the diagram still makes sense when exported as monochrome SVG or PNG
